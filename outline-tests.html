Method 4: Current Three.js Approach (Scaled Geometry)

Current Implementation: Uses two 3D text meshes - one scaled larger for outline, one normal size for fill. This is what's currently in index.html.
// Current approach from index.html const outlineGeometry = textGeometry.clone(); outlineGeometry.scale(outlineScale, outlineScale, 1); const outlineMesh = new THREE.Mesh(outlineGeometry, outlineMaterial); const fillMesh = new THREE.Mesh(textGeometry, fillMaterial); const textGroup = new THREE.Group(); textGroup.add(outlineMesh); textGroup.add(fillMesh);

‚úÖ Pros

  • True 3D geometry
  • Works with Three.js TextGeometry
  • No texture conversion needed

‚ùå Cons

  • Not a true outline (scaled geometry)
  • Outline width inconsistent on curves
  • Double the geometry (performance)
  • Potential z-fighting issues

Best Approach: Canvas ‚Üí Texture

For the mantra app, the best approach would be to:

  1. Generate text with outlines using Canvas (Method 3)
  2. Use as texture on PlaneGeometry instead of TextGeometry
  3. Position planes in the ring exactly like current syllables
  4. Always face camera (billboard effect - already implemented for TAM)

Why this works best:

  • ‚úÖ Perfect, consistent outlines
  • ‚úÖ Much better performance (plane vs extruded geometry)
  • ‚úÖ Full control over outline width and color
  • ‚úÖ Already have billboard code from TAM character
  • ‚úÖ Easier to implement than current dual-geometry approach
